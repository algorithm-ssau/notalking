# v0.3

## Changes

- Reworked the note editor to keep block-local optimistic state, preserve caret and selection ranges more reliably, and keep bold or italic formatting in the existing chunk-based storage model.
- Added inline note-title rename in the editor header with a dedicated note-level Core update route and matching web API support.
- Improved drag and drop with clearer insertion feedback, stable local reordering, and safer block movement behavior.
- Expanded the assistant panel with used-note summaries, preview-then-apply note writing for create or edit flows, and a stop button that cleanly interrupts streaming responses.
- Extended Core, MCP, and Intelligence contracts with explicit note update capabilities for rename, append, and replace operations.
